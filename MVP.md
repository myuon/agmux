# agmux — AI Agent Multiplexer

## 1. プロジェクトの目的

複数のAI agentセッションを同時に実行・監視・制御するための統合管理ツール。
1つのダッシュボードから、このPC上で動いているあらゆるagentの状態を把握し、介入できるようにする。

## 2. 目標

- 複数のClaude Codeセッションをtmux上で同時管理する
- デーモンが各セッションを定期巡回し、LLMで状態を判断して自律的に介入する
- Web UIダッシュボードで全セッションの状態をリアルタイムに一覧・操作できる
- 将来的にはCodex等の他agentや、リモートサーバー上のセッションも管理対象とする

## 3. Non-Goals（今回やらないこと）

- Claude Code以外のagent対応
- リモートサーバー上のセッション管理
- ユーザー認証・マルチユーザー対応
- セッション間の依存関係管理・オーケストレーション
- モバイル対応

## 4. 対象ユーザー

- ローカルPCで複数のClaude Codeセッションを並行して回す開発者（自分自身）

## 5. 技術スタック

| カテゴリ | 選定 | 理由 |
|---------|------|------|
| バックエンド | Go | single binary、デーモン向き |
| フロントエンド | TypeScript + React + Vite | 軽量・高速 |
| HTTPサーバー | net/http + chi | 標準ライブラリベース |
| リアルタイム通信 | WebSocket (gorilla/websocket) | 状態更新のプッシュ |
| データ永続化 | SQLite (mattn/go-sqlite3) | 組み込み、single binary対応 |
| LLM連携 | Anthropic API (Claude Sonnet) | デーモンの状態判断用 |
| セッション管理 | tmux (CLI経由) | プロセス維持の標準ツール |
| テスト | Go testing + testify / Vitest | 各言語の標準的なテストツール |
| ビルド | Go embed + esbuild | フロントをGoバイナリに同梱 |

最終成果物は **single binary 1個** にビルドされる。

## 7. MVP機能一覧

### 7.1 セッション管理

- [ ] 新規Claude Codeセッションの作成（tmux session + claude code起動）
- [ ] セッション一覧の取得
- [ ] セッションの停止・削除
- [ ] セッションへのテキスト送信（tmux send-keys）

### 7.2 状態監視

- [ ] tmux capture-paneによるセッション画面内容の取得
- [ ] ログファイルの監視・取得
- [ ] セッション状態の分類（実行中 / 入力待ち / エラー / 完了）

### 7.3 デーモン

- [ ] 設定可能な間隔（デフォルト30秒）での定期巡回
- [ ] Claude Sonnet APIで各セッションの状態を判断
- [ ] 判断に基づく自律的アクション:
  - 承認待ち（permission prompt）→ 自動承認送信
  - エラー発生 → リトライ指示の送信
  - ユーザー判断が必要 → エスカレーション通知
  - 正常稼働中 → 何もしない
- [ ] デーモンのアクション履歴の記録

### 7.4 Web UIダッシュボード

- [ ] 全セッション一覧表示（状態・最終更新時刻）
- [ ] 各セッションの詳細表示（画面キャプチャ内容、ログ）
- [ ] セッションの新規作成フォーム
- [ ] セッションへの手動メッセージ送信
- [ ] デーモンのアクション履歴表示
- [ ] WebSocketによるリアルタイム更新

### 7.5 設定

- [ ] 巡回間隔の設定
- [ ] LLMモデルの設定
- [ ] 自動承認の有効/無効
- [ ] 設定ファイル（YAML or TOML）

## 8. ユーザーフロー（MVP）

1. `agmux` を起動する → デーモン + Webサーバーが起動
2. ブラウザで `http://localhost:PORT` を開く
3. ダッシュボードで「新規セッション作成」→ プロジェクトパスと初期プロンプトを入力
4. tmux上でClaude Codeセッションが起動し、ダッシュボードに表示される
5. デーモンが定期的に各セッションを巡回
6. 承認待ちやエラーを検知 → 自動対応 or ユーザーに通知
7. ダッシュボードからセッションの状態確認・手動介入が可能

## 9. Acceptance Criteria

1. `agmux` コマンド1つでデーモン+Webサーバーが起動する
2. Web UIから新規Claude Codeセッションを作成できる
3. 作成したセッションがtmux上で実際に動作している
4. ダッシュボードに全セッションの状態がリアルタイム表示される
5. デーモンが承認待ちのセッションを検知し自動で承認を送信できる
6. デーモンがエラー状態のセッションを検知しリトライ指示を送信できる
7. ユーザーの判断が必要な場合にダッシュボード上で通知される
8. Web UIからセッションにテキストを手動送信できる
9. single binaryとしてビルドできる
10. デーモンのアクション履歴がダッシュボードで確認できる

## 10. 検証戦略

- **進捗検証**: 各機能実装後に `go run` + ブラウザで動作確認
- **達成検証**: 上記Acceptance Criteriaをチェックリストで全項目確認
- **漏れ検出**: 実際に5セッション同時起動して一連のフローを通して操作する
